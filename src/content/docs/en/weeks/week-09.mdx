## Theory

### The Independence Principle of the QA Agent

The QA Agent **never shares context with the Coder Agent**. Two agents sharing the same context share the same biases, making independent verification impossible.

<Aside type="note" title="QA Agent Independence">
Even if the Coder has determined "this code is correct," the QA Agent verifies independently — seeing **only the code and tests**, with no knowledge of the Coder's reasoning process.
</Aside>

### QA Agent Implementation

```python
# qa_agent.py
import subprocess
import anthropic
from pathlib import Path

class QAAgent:
    def __init__(self):
        self.client = anthropic.Anthropic()

    def run_tests(self, test_dir: str) -> dict:
        """Run pytest and parse results"""
        result = subprocess.run(
            ["python", "-m", "pytest", test_dir, "-v", "--tb=short", "--json-report"],
            capture_output=True, text=True
        )
        return {
            "passed": result.returncode == 0,
            "output": result.stdout,
            "errors": result.stderr
        }

    def code_review(self, diff: str) -> str:
        """Code review via Claude"""
        response = self.client.messages.create(
            model="claude-opus-4-6",
            max_tokens=2048,
            system="""You are a senior software engineer.
Review the code diff and check for:
1. Logic errors
2. Unhandled edge cases
3. Security vulnerabilities
4. Performance issues
5. Missing tests

Output: JSON {"approved": bool, "issues": [...], "suggestions": [...]}""",
            messages=[{"role": "user", "content": f"Code review request:\n{diff}"}]
        )
        return response.content[0].text

    def review_pr(self, pr_diff: str, test_dir: str) -> dict:
        """Full PR validation"""
        test_result = self.run_tests(test_dir)
        review_result = self.code_review(pr_diff)

        return {
            "tests_passed": test_result["passed"],
            "test_output": test_result["output"],
            "code_review": review_result,
            "approved": test_result["passed"] and "approved: true" in review_result.lower()
        }
```
