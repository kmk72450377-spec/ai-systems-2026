## 통합 테스트 체크리스트

<Steps>
1. **기능 테스트** — 각 에이전트 개별 기능 100% 통과

2. **통합 테스트** — 전체 파이프라인 end-to-end 실행 3회 이상 성공

3. **스트레스 테스트** — 연속 10개 태스크 처리 시 안정성 확인

4. **실패 복구 테스트** — 의도적 실패 주입 후 자동 복구 확인

5. **성능 측정** — 처리 시간, 토큰 사용량, 성공률 기록
</Steps>

## 발표 자료 구성

**권장 발표 구조 (15분)**:

1. **문제 정의** (2분) — 어떤 문제를 해결하려 했나?
2. **아키텍처** (3분) — 어떻게 설계했나? 다이어그램 필수
3. **라이브 데모** (5분) — 실제 에이전트 실행 (사전 녹화 권장)
4. **결과 및 성과** (3분) — 성능 수치, 인상적인 결과
5. **배운 점** (2분) — 가장 어려웠던 점, 다시 한다면?

<Aside type="tip" title="데모 준비">
라이브 데모는 네트워크 문제, API 오류 등 예측 불가한 실패가 발생할 수 있다. 반드시 **사전 녹화본을 백업**으로 준비하라.
</Aside>
